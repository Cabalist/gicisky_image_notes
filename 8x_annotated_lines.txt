0x380000 is actually 56B which is 448b.  This leaves 64b or 8B.  There are 4 bytes at the end and 4 bytes at the beginning.

It is decremented everytime a byte is added to the data
0x120000 .. 0x380000 is straightforward but for values less than 0x120000 it switches to a different format
This is mapped in the table above.  It probably is more complicated than this but we can't see any examples here.


SO parsing-wise if I gobble 4 zeros I need to gobble 2 more characters to get the value I'm looking for.
